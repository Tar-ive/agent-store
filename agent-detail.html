Installation & Setup

Requirements

  • Node.js 18+ and npm
  • Vercel CLI installed globally (`npm install -g vercel`)
  • Valid `VERCEL_TOKEN`, `VERCEL_PROJECT`, and `VERCEL_ORG` environment values
  • Access to the LangChain.md repo (or fork) with deploy permissions

Clone + Prepare

git clone https://github.com/tarive/langchain-md.git
cd langchain-md
npm install -g vercel

Environment Setup

# Required environment variables
export VERCEL_TOKEN="your-vercel-token"
export VERCEL_PROJECT="langchain-md"
export VERCEL_ORG="your-team-slug"

# Optional: log file from previous runs
rm -f ops-agent.log

Ops Playbook

Ops Field Agent wraps the LangChain.md deployment workflow into a reproducible CLI. Plug in your Vercel credentials and it links the repo, enables Web Analytics, and ships a production build.

Execute Run

cd langchain-md
export VERCEL_TOKEN=your-token
export VERCEL_PROJECT=langchain-md
export VERCEL_ORG=langchainmd
./tools/ops-agent/run_ops_agent.sh

Environment Variables

Variable Description
VERCEL_TOKEN Personal or team token with deploy + analytics scope.
VERCEL_PROJECT Vercel project slug (e.g. langchain-md).
VERCEL_ORG Team/organization slug used for --scope.

What the Agent Does

  • `vercel link --yes` to bind the repo to the configured project.
  • `vercel analytics enable --yes` to guarantee Web Analytics stays active.
  • `vercel deploy --prod --yes` to ship the current working tree.
  • Append run output to `ops-agent.log` for incident correlation.

Example Log Snippet

[ops-agent] Using project=langchain-md, org=langchainmd
[ops-agent] Linking local repo to Vercel project...
✔ Linked to langchain-md (production)
[ops-agent] Enabling Vercel Web Analytics (idempotent)...
✔ Analytics already enabled
[ops-agent] Deploying to production...
✔ Deployment complete https://langchain-md.vercel.app
